## Feature Slices Branch (Started: fre 3 apr. 2026)

## Codebase Patterns
- Convex file-based routing: `convex/elevators/crud.ts` -> `api.elevators.crud.get`
- Both apps have `src/convex-api.d.ts` files that provide TypeScript types for `@convex/_generated/api` — these must be updated when Convex API structure changes
- `@convex` Vite alias is configured in each app's `vite.config.ts`, NOT in tsconfig paths
- `autoAddSuggestedValues` is imported by `convex/importsInternal.ts` — update import path when moving helpers
- Pre-existing typecheck errors: checkbox module was missing — FIXED in US-004 (Checkbox component added to packages/ui)
- Convex codegen (`npx convex codegen`) uploads code to deployed instance — be careful not to run with wrong files checked out
- `maintenanceCompanies` was renamed to `companies` in the maintenance module (shorter name since module provides context)
- TanStack Start with file-based routing: routes in `apps/admin/src/routes/_authenticated/`
- Route tree auto-generated in `routeTree.gen.ts` (gitignored) — regenerated by dev server / vite build
- shadcn/ui components imported from `@elevatorbud/ui/components/ui/*`
- Convex API imported as `import { api } from "@convex/_generated/api"`
- When a route has `validateSearch`, all `<Link to=...>` pointing to it must include `search` prop
- CMS form types (CmsPage, CmsSection) defined in `apps/admin/src/components/cms/startsida-form.tsx`

- **Learnings for future iterations:**
  - The `convex-api.d.ts` files in each app are the actual source of TypeScript types for `@convex/_generated/api` — the generated `api.d.ts` in convex/_generated is NOT used by tsc for the apps
  - Running `npx convex codegen` with wrong files checked out will corrupt the deployed Convex state and regenerate wrong types
  - Convex function names in subdirectories drop the "modernization" prefix since the module name provides context (e.g., `modernizationTimeline` -> `modernization.timeline`)
  - `helpers.ts` exports plain TS functions (not query/mutation), so it doesn't appear in the public API
