## Codebase Patterns

### TanStack Query + Convex Integration
- Package: `@convex-dev/react-query` provides `ConvexQueryClient`, `convexQuery`, `convexAction`
- Package: `@tanstack/react-router-ssr-query` provides `setupRouterSsrQueryIntegration` (PRD references old name `@tanstack/react-router-with-query` / `routerWithQueryClient` — the API was renamed)
- Setup: ConvexReactClient → ConvexQueryClient(convexClient) → QueryClient with hashFn/queryFn → connect → setupRouterSsrQueryIntegration
- `setupRouterSsrQueryIntegration` wraps router's `Wrap` with `QueryClientProvider` (composes with existing Wrap)
- Router `Wrap` has `ConvexProvider` for SSR; root route adds `ConvexProviderWithClerk` for client auth (inner overrides outer)
- `convexQuery(api.foo.bar, args)` returns query options for `useQuery`/`useSuspenseQuery`
- `convexQuery(api.foo.bar, "skip")` returns options with `enabled: false`
- Mutations/actions stay on `convex/react` — only queries migrate to TanStack Query

### useSuspenseQuery Type Workaround
- `convexQuery()` returns a union type (skip vs non-skip) that `useSuspenseQuery` rejects
- Workaround: extract `queryKey` and `staleTime` individually from the convexQuery result
- The default `queryFn` on the QueryClient handles the actual fetching
- Pattern: `const opts = convexQuery(api.foo, {}); useSuspenseQuery({ queryKey: opts.queryKey, staleTime: opts.staleTime })`
- For `ensureQueryData` in loaders: pass `convexQuery()` directly (its type is accepted)

### SSR Auth Pattern
- Root route `beforeLoad` fetches Clerk token via `createServerFn` and sets it on `convexQueryClient.serverHttpClient?.setAuth(token)`
- This enables authenticated Convex queries during SSR (e.g. `ensureQueryData` in loaders)
- `serverHttpClient` is only present during SSR; on client it's `undefined` (safe with optional chaining)
- Client-side auth handled by `ConvexProviderWithClerk` in root route component

### Dependent/Conditional Queries
- Primary query: `useSuspenseQuery` with queryKey/staleTime extraction
- Dependent query: `useQuery` from `@tanstack/react-query` with `convexQuery(api.foo, condition ? args : "skip")` — spread directly
- The "skip" variant sets `enabled: false` automatically, so no manual `enabled` flag needed
- `useQuery` (not `useSuspenseQuery`) is correct here because the query may not fire
