## Codebase Patterns

- **Self-referential FK in Drizzle**: Use `(): any =>` for the references callback to avoid circular type issues. Use `relationName` on both sides of self-referential relations.
- **DB-level constraints for hierarchy**: Use a BEFORE INSERT OR UPDATE trigger (not a check constraint) when enforcement requires querying other rows. Drizzle doesn't generate triggers, so add them manually to the migration SQL.
- **Platform mismatch (macOS → Linux container)**: Must `npm install -g pnpm` and `CI=true pnpm install --no-frozen-lockfile` to get working node_modules. esbuild also needs the correct platform binary — install matching version via `npm install @esbuild/linux-arm64@<version>` in the package dir, then use `ESBUILD_BINARY_PATH` env var.
