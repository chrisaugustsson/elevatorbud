## Codebase Patterns
- TanStack Start with file-based routing: routes in `apps/admin/src/routes/_authenticated/`
- Route tree auto-generated in `routeTree.gen.ts` (gitignored) — regenerated by dev server / vite build
- Sidebar nav defined as `const xxxNav = [...] as const` arrays in `app-sidebar.tsx`, with `SidebarGroup > SidebarGroupLabel > SidebarGroupContent > SidebarMenu` structure
- Icons from `lucide-react`
- Active state: `pathname.startsWith(item.href)` for grouped nav, exact match for main nav
- Typecheck: `npx tsc --noEmit --project apps/admin/tsconfig.json` from repo root
- shadcn/ui components imported from `@elevatorbud/ui/components/ui/*`
- Convex API imported as `import { api } from "@convex/_generated/api"`
- When a route has `validateSearch`, all `<Link to=...>` pointing to it must include `search` prop — TanStack Router enforces this at type level
- shadcn components installed via `pnpm dlx shadcn@latest add <name>` from `packages/ui/` directory
- Tabs component: `@elevatorbud/ui/components/ui/tabs` — Tabs, TabsList, TabsTrigger, TabsContent
- CMS form types (CmsPage, CmsSection) defined in `apps/admin/src/components/cms/startsida-form.tsx` — reuse for other page forms
- `@convex/_generated/dataModel` is NOT declared in `convex-api.d.ts` — define CMS types inline instead of importing `Doc<"pages">`
- CMS sections accessed via `page.sections.find(s => s.type === "xxx")` — stats maps value→item.title, label→item.description
- CMS page forms follow pattern: FormState type → emptyState() → pageToFormState(page) → update functions → Card-per-section JSX; import CmsPage from startsida-form.tsx
- Webbplats route conditionally renders page forms by slug in PageTab component
- Toast notifications: `sonner` package, `toast.success()`/`toast.error()`, Toaster in __root.tsx
- Form save pattern: each form has `onSave: (sections: CmsSection[]) => void` + `isSaving: boolean` props; PageTab handles create/update mutation logic
- Create vs update: check if `useQuery` result has `_id` — if page exists, call `updatePage` with `page._id`; if null, call `createPage` with slug
- Page-level fields (like `published`) managed in PageTab with useEffect sync from query data, passed in mutation payloads alongside sections
