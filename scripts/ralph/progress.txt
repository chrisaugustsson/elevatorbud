## Codebase Patterns
- Unicode in JSX: Source files had `\uXXXX` escape sequences in JSX text content and string attributes. These are NOT decoded in JSX text — must use actual UTF-8 characters instead.
- Feature components live in `apps/*/src/features/<domain>/components/`
- Both admin and client apps share similar patterns — changes to shared domain labels may need updating in both apps
- TanStack Router: use `useRouter()` + `router.history.back()` for back navigation instead of hardcoded `<Link>` paths
- Debounce pattern: use `useEffect` with `setTimeout`/`clearTimeout` for debouncing search inputs that drive Convex queries — clear immediately when input is empty for instant feedback
- Sidebar state: use `useSidebar()` hook from `@elevatorbud/ui/components/ui/sidebar` — provides `state` ("expanded"|"collapsed"), `toggleSidebar()`, `isMobile`
- Collapsed sidebar: use `group-data-[collapsible=icon]:hidden` to hide elements when sidebar is in icon-only mode, `group-data-[collapsible=icon]:ml-0` to adjust spacing
- TanStack Form: `useForm` return type is too complex for `ReturnType<typeof useForm<...>>` — inline form fields in each dialog component instead of extracting shared form components
- Table actions column: use `columnHelper.display({ id: "actions", ... })` for non-data columns (no accessor)
- Confirmation dialogs: use Dialog (not AlertDialog — doesn't exist in UI package) with destructive Button variant for dangerous actions
- Current user identity: `useUser()` from `@elevatorbud/auth` gives `user.id` (Clerk user ID) — compare with `clerk_user_id` field in user records
- DropdownMenu: available at `@elevatorbud/ui/components/ui/dropdown-menu` — use DropdownMenu, DropdownMenuTrigger, DropdownMenuContent, DropdownMenuItem, DropdownMenuSeparator
- Convex optional field clearing: `v.optional()` args with `if (field !== undefined)` pattern means `undefined` = "don't change". To clear a field, add explicit logic (e.g., clear org when role is admin) rather than relying on `undefined`
- Optional URL search params: Avoid `validateSearch` on routes that are widely linked — it makes `search` required on ALL Link components. Instead use `useState(getInitialFromURL)` + `window.history.replaceState` for optional URL param persistence
- Command palette: `cmdk` package + `@elevatorbud/ui/components/ui/command` — use `CommandDialog` for modal command palette, supports built-in keyboard navigation (arrow keys, Enter, Escape)
- TanStack Router navigation to non-existent routes: cast `to` param as `string` to bypass type checking for routes that will be created later
- TanStack Router sibling routes: When adding `foo.$id.tsx` alongside `foo.tsx`, rename `foo.tsx` to `foo.index.tsx` so both are siblings (not parent-child). Without this, the router treats `foo.tsx` as a layout requiring `<Outlet />`
- Route tree generation: `routeTree.gen.ts` is auto-generated by the Vite dev server (`tanstackStart()` plugin) — must briefly start dev server to regenerate after adding new route files
