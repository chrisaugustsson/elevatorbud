# Ralph Progress Log
Started: ons  2 apr. 2026

## Codebase Patterns
- Root tsconfig.json is a base config — individual packages extend it with their own tsconfig.json
- Convex aggregation queries: `.collect()` + JS reduce/filter — no native aggregation functions, use `.withIndex()` for org-scoped queries when possible
- Turbo pipelines: build, dev, lint, typecheck — use `pnpm turbo run <task>` to run across all packages
- pnpm workspaces: apps/* and packages/* — all new apps go in apps/, shared code in packages/
- .gitignore covers: node_modules, dist, .turbo, .vinxi, .output, .wrangler, .env, .env.local, *.tsbuildinfo, routeTree.gen.ts
- TanStack Start apps use vite.config.ts with `@tanstack/react-start/plugin/vite` + `@cloudflare/vite-plugin` + `@vitejs/plugin-react`
- App tsconfig.json must NOT set `rootDir: "src"` when `include` also contains `vite.config.ts` — use just `include: ["src", "vite.config.ts"]`
- Route tree files (routeTree.gen.ts) are auto-generated by the TanStack Start Vite plugin during build/dev — do not create manually
- For Cloudflare Workers: use `@cloudflare/vite-plugin` (not Nitro) — this is the recommended approach per TanStack Start docs
- App ports: admin=3001, client=3002, landing=3000
- Native build dependencies (esbuild, sharp, workerd) must be approved in root package.json `pnpm.onlyBuiltDependencies`
- Shared UI package: `@elevatorbud/ui` in packages/ui/ — import components via `@elevatorbud/ui/components/ui/button` etc.
- Tailwind CSS v4 uses CSS-based config (`@import "tailwindcss"`, `@theme inline`) — NO tailwind.config.js
- All apps include `@tailwindcss/vite` in vite.config.ts plugins array
- App CSS entry: `src/styles/app.css` imports `@elevatorbud/ui/styles/globals.css` — loaded via `?url` import in `__root.tsx`
- shadcn/ui components installed via `npx shadcn@latest add <component>` from packages/ui/ directory
- components.json exists in both packages/ui/ (for adding components) and each app (for CLI compatibility)
- `cn()` utility: import from `@elevatorbud/ui/lib/utils` — uses clsx + tailwind-merge
- Stub packages pattern: package.json (name, exports, typecheck script), tsconfig.json (extends root, outDir: dist), src/index.ts
- Package exports for non-UI packages: `".": "./src/index.ts"` and `"./*": "./src/*.ts"` for subpath imports
- Convex 1.34.1 installed as root devDependency — convex/ directory lives at monorepo root, NOT inside an app
- convex.json at root configures Convex (default functions dir is `convex/`)
- convex/tsconfig.json is standalone (does NOT extend root) — typechecked separately via `tsc --noEmit -p convex/tsconfig.json`
- Root `typecheck` script chains: turbo typecheck (all packages) + tsc for convex/
- Turbo root tasks use `//#taskname` syntax in turbo.json — `dev:convex` runs `npx convex dev` alongside app dev servers
- CMS backend: convex/cms.ts has getPage, listPages, createPage, updatePage — sectionValidator defines section shape
- Admin app sidebar: follow existing sidebar patterns for new nav items
