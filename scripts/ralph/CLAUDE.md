# Ralph Agent Instructions

You are an autonomous coding agent working on the Elevatorbud project — a monorepo for elevator management (hisskompetens) built with TanStack Start, Convex, and Tailwind CSS.

## Branching

**ALWAYS branch off from `dev`.** Never branch from `main` or any other feature branch.

```bash
git fetch origin dev
git checkout -b ralph/<branch-name> origin/dev
```

If the PRD specifies a `branchName`, create that branch from `origin/dev`. If the branch already exists, rebase it onto `origin/dev` before starting work to avoid merge conflicts with other feature branches.

## Project Structure

This is a pnpm monorepo:

- `apps/admin/` — Admin dashboard (TanStack Start, file-based routing)
- `apps/client/` — Client portal (TanStack Start, file-based routing)
- `apps/landing/` — Landing page
- `convex/` — Backend (Convex functions, file-based routing)
- `packages/ui/` — Shared UI components (shadcn/ui based)
- `packages/utils/` — Shared utilities (excel import, export, etc.)
- `packages/auth/` — Auth package

### Key Conventions

- Routes live in `apps/*/src/routes/_authenticated/`
- Route tree (`routeTree.gen.ts`) is auto-generated and gitignored — regenerated by dev server
- Feature code is organized in `apps/*/src/features/<domain>/` with hooks/, components/, and types
- Shared/reusable components go in `packages/ui/src/components/ui/`
- Convex API is imported as `import { api } from "@convex/_generated/api"`
- Both apps have `src/convex-api.d.ts` that provides TypeScript types — update these when Convex API structure changes
- `@convex` Vite alias is configured in each app's `vite.config.ts`, NOT in tsconfig paths
- shadcn/ui components: `@elevatorbud/ui/components/ui/*`
- Icons: `lucide-react`

### Convex Backend

- File-based routing: `convex/elevators/crud.ts` → `api.elevators.crud.get`
- `convex/elevators/` contains domain modules: `crud.ts`, `listing.ts`, `analytics.ts`, `modernization.ts`, `maintenance.ts`, `helpers.ts`
- `helpers.ts` exports plain TS functions (not query/mutation), so it doesn't appear in the public API
- `npx convex codegen` uploads code to the deployed instance — be careful not to run with wrong files checked out
- When modifying Convex function structure, update `convex-api.d.ts` in both apps

### Excel Import System

- Import logic lives in `packages/utils/src/excel-import/` (split into multiple files)
- Frontend feature slices in `apps/admin/src/features/import/`
- Uses column auto-mapping flow: file parse → mapping step → preview → confirm
- Key exports: `autoMapSheet`, `autoMapColumns`, `getSheetData`, `parseExcelImportWithMapping`, `TARGET_FIELDS`
- The `confirmImport` action uses parallel arrays for org matching (Convex doesn't allow non-ASCII keys in objects)
